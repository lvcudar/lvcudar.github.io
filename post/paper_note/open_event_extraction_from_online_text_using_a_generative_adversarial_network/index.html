Read Paper 《Open Event Extraction from Online Text using a Generative Adversarial Network》

2019-12-07
Paper Note NLP

Contents

URL: https://www.aclweb.org/anthology/D19-1027/

简介

EMNLP 2019 主要面对长文本事件复杂的情况，用对抗网络的方法实现开放事件抽取(发现其中第一个事件)，比之前的方法更快速，提升明显。

数据集

本文的数据集不怎么完善，都是从网上爬取，然后标注的，比较小而且难以获得同样的数据集。

  1. FSD(First story Detection): 出自一篇论文但是论文中的下载链接已经失效了。找到了相关代码
  2. Twitter: 临时从Twitter上爬的1000条推文标注20个事件，感觉就是爬取时间不同的FSD数据集。
  3. Google: GDELT的一部分，用30个流行事件筛选出了其中11909篇文章。

模型

不太理解Fig3的含义。

模型由三部分组成:

  1. 文档表示由传统方法得到，由实体向量、地点向量、关键词向量、日期向量拼接而成,文中举例了实体方面的计算。 \( tf^e_{i,doc} = \frac{n^e_{i,doc}}{\sum_{v_e} n^e_{v^e, doc}} \) \( idf^e_i = \log \frac{|C^e|}{|C^e_i|} \) \( tf\text{-}idf^e_{i,doc} = tf^e_{i,doc} * idf^e_i \) \( d^e_{r,i} = \frac{tf\text{-}idf^e_{i,doc}}{\sum_{v_e} tf\text{-}idf^e_{v_e,doc}} \) 其中，i表示第i个实体，也是维度；n表示出现计数；d表示相关性；也就是说实体向量是文章和每个实体的相关性(所以实体不能太多?)。

  2. 生成器是用神经网络来从文档中选择相应的实体、日期、地点等。

  3. 鉴别器是多层全联接网络。

方法

首先事件被描述为四元组<实体、地点、关键词、日期>。 相关工作介绍中，之前的研究不多。其中对于短文本的事件抽取假设所有单词来自一个事件有局限，而长文本多事件建模中吉布斯采样和贝叶斯概率图模型速度慢。 这里使用对抗训练的方法，用生成器学习文档分布到事件的变换，用鉴别器判断一个事件四元组来自生成器还是真实事件。

实验

实验和传统的K-Means聚类、还有作者自己之前两种贝叶斯图模型 LEM DPEMM 进行了比较。准确率有稳定的提高。

相对于吉布斯采样，GAN能并行充分发挥GPU的优势，充分并行，速度优势明显。

思考

同作者17年还有一篇同一方向的论文，也未公开代码，数据集同样小且不易获取。 数据集种的文本是用流行事件筛选的，感觉会有偏差。 不直接对比GT，而是和前一步的概率向量做对抗训练，是一个不错的GAN构造上的创新。
